# FNIRSI DPS-150

## Identity

- **Vendor:** FNIRSI
- **Model:** DPS-150 programmable DC bench supply
- **Canonical slug:** `dps150`
- **Status:** Active retail product.
- **Interface:** USB-C → USB serial (CDC-ACM). 115200 baud, RTS/CTS flow control, custom binary framed protocol (0xF1 host→device, 0xF0 device→host). No Digilent / NI runtime required.

## Capabilities

| Parameter | Spec |
|---|---|
| Voltage range | 0 – 30 V programmable, 10 mV setpoint resolution |
| Current range | 0 – 5 A programmable, 10 mA limit resolution |
| Output on/off | Software-controlled, separate from setpoint |
| Telemetry | Voltage / current / power returned from `/psu/measurements` |
| Channels | 1 (single-output supply) |
| Safety | Over-voltage / over-current lockout configurable on the front panel |

## Pin layout

- **USB-C** (rear): host control. Appears as `/dev/tty.usbmodem*` (macOS) or `/dev/ttyACM0` (Linux).
- **Output terminals** (front): banana jacks, + / −.

The DPS-150 is a single-channel supply — no per-channel index in any API call.

## Safety limits

- Scopecreep-side clamp: max 5.0 V, max 1.0 A. Refuse user requests outside this envelope regardless of the device's own 30 V / 5 A range.
- Always set the current limit **before** enabling the output, so an unexpected short doesn't fault the DUT.
- Disable the output in a `finally:` block on any test script that touches `/psu/output`.

## Common operations

All calls go to the agent backend on `http://127.0.0.1:8000`. The DPS-150 routes live under `/psu/*` (distinct from the AD PSU routes — the DPS-150 routes use serial port + volts/amps payloads rather than channel index).

### Connect to the DPS-150

```python
import requests
API = "http://127.0.0.1:8000"
# macOS: "/dev/tty.usbmodem2101"   Linux: "/dev/ttyACM0"
r = requests.post(f"{API}/psu/connect", json={"port": "/dev/ttyACM0"})
r.raise_for_status()
print(r.json())  # {"status": "connected", "port": "..."}
```

### Set voltage + current limit, enable output

```python
requests.post(f"{API}/psu/current", json={"amps": 0.5}).raise_for_status()   # limit FIRST
requests.post(f"{API}/psu/voltage", json={"volts": 3.3}).raise_for_status()
requests.post(f"{API}/psu/output",  json={"enabled": True}).raise_for_status()
```

### Measure

```python
m = requests.get(f"{API}/psu/measurements").json()
# {"voltage": 3.298, "current": 0.112, "power": 0.370}
```

### Disable output + disconnect

```python
requests.post(f"{API}/psu/output",  json={"enabled": False}).raise_for_status()
requests.delete(f"{API}/psu/disconnect").raise_for_status()
```

### Voltage sweep with per-step measurements

```python
import time, requests
API = "http://127.0.0.1:8000"
try:
    requests.post(f"{API}/psu/current", json={"amps": 0.5}).raise_for_status()
    readings = []
    for v in [0.0, 0.5, 1.0, 1.5, 2.0, 2.5, 3.0, 3.3]:
        requests.post(f"{API}/psu/voltage", json={"volts": v}).raise_for_status()
        requests.post(f"{API}/psu/output", json={"enabled": True}).raise_for_status()
        time.sleep(0.2)
        readings.append(requests.get(f"{API}/psu/measurements").json())
    print(readings)
finally:
    requests.post(f"{API}/psu/output", json={"enabled": False})
    requests.delete(f"{API}/psu/disconnect")
```

## Known gotchas

- **Port name discovery** — generated test code should prompt the user for the port, or call a helper that scans `/dev/tty.usbmodem*` / `/dev/ttyACM*`. Do NOT hardcode.
- **Set current limit before voltage+output.** Setting voltage while output is off is harmless; enabling output with a high current limit against a shorted DUT isn't.
- **Telemetry arrives unsolicited.** The driver returns cached values from `/psu/measurements`. After setting a new voltage, allow 100–200 ms before reading.
- **Serial port held by driver process.** If a script errors out mid-run, the next `/psu/connect` may 503 with "resource busy" — `/psu/disconnect` clears it; restart the sidecar as a fallback.
- **The DPS-150 routes are distinct from AD PSU routes.** AD uses `/psu/set` + `/psu/enable` with `{"ch": 1, ...}`; DPS-150 uses `/psu/voltage` + `/psu/current` + `/psu/output` with volts/amps/enabled payloads. Check `/psu/state` vs `/status` to see which backend is live.
